PPDRIVE - an integratable and secure file manager

PPDRIVE is a powerful, standalone file manager written in Rust, designed for seamless integration into other applications via a RESTful API.

Client Routes

Client Routes are accesible only to a verifiable PPDRIVE Client. A client is an external application capable of authorizing users and managing assets on PPDRIVE.

To access these routes, you need to provide your client's token by adding the token to x-ppd-client header of your requests. You can generate a client token by running ppdrive create_client {client_name} or via the Admin UI.

IMPORTANT! Client tokens give administrative access to anyone who possesses it. DO NOT SHARE YOUR CLIENT TOKEN. This means you should ONLY access client routes from your application server.

If your client token is compromised (or for any other reason), you can regenerate a client token by running ppdrive new_token {client_id}.

Register User

Register a new user.
If you're connecting PPDRIVE to an exisiting app (likely a reason to use clients), it's perfect to call this route before/after you've registered your user and bind the returned id to your app's user. See responses.200

Authorizations:
clientKey
Request Body schema: application/json
required

Options for creating a new user

role
required
string
Enum: "Basic" "Manager"

The user's role

partition
string

Create a dedicated partition for the user named by value provided. If this is not specified, user's assets will be stored in application's root partition. Partition name must be unique for every user. If the value is already take, this call will return an error.

partition_size
integer

Max size (in bytes) to be assiged to the user's partition. This should be set only if partition is set. Otherwise, it would be ignored.

Responses

A local demo server

http://localhost:5000/client/user/register

Request samples

  • Payload
Content type
application/json
{
  • "role": "Creator",
  • "partition": "prodbyola"
}

Response samples

  • 200
Content type
appplication/json
{
  • "token": "string",
  • "exp": 0
}

Login User

Authenticate a user and get their JWT. password field is not required for this route.

Authorizations:
clientKey
Request Body schema: application/json
required

Options for creating a new user

id
required
string

This could be the user's id or email address.

password
string

The user's password. Password is optional if you're not coming though a Client Route.

exp
integer

Optional expiration period for user's token (in seconds). This defaults to 5 hours if not set. We're working on making the default value configurable.

Protected Routes

Protected Routes are accesible only to a user with Manage or Admin role. The user_role is assigned when a client or admin creates a new user.

The Protected must be logged in before they can access this route. Call client's login route to get a user's Bearer Token and pass the token to Authorization header of your request in the format Bearer {token}.

Create or Upload Asset

Create a new asset. If the asset already exists, it will be updated.

It's important for PPDRIVE to keep record of every asset and their ancestors in the filesystem. Therefore, this route traverses the asset path provided in options, if no record is found for any of its parent within the file hierarchy, we create the record and the respective asset. You can disable this behavior by settting options.create_parents to false.

Authorizations:
userAuth
Request Body schema: application/multipart
required

Options for creating the new asset

path
string,

The relative path where we save user's asset.

asset_type
string
Enum: "File" "Folder"

The type of asset - whether it's a file or folder

public
boolean

Asset's visibility.
Public assets can be accessed by everyone. Private assets can only be accessed by users with Read permission.

custom_path
string

Set a custom path for your asset instead of the one auto-generated from path. This useful if you'd like to conceal your original asset path. custom_path is unique to each asset and is available if no other asset is already using it in the entire app.

Your original asset path makes asset url look like this https://mydrive.com/images/somewhere/my-image.png/. Using custom path, you can conceal the original path: https://mydrive.com/custom-path

create_parents
boolean

Whether to traverse path and create parent assets if they do do not exist.

Array of objects (AssetSharing)

Users to share this asset with. This can only be set if public option is false. Otherwise, it'll be ignored. Also, you cannot share asset with asset owner. This will also be ignored.

Responses

A local demo server

http://localhost:5000/asset

Response samples

  • 200
Content type
appplication/json
{
  • "token": "string",
  • "exp": 0
}

Get Asset

Serve an asset (file or folder) from the provided asset path.

Authorizations:
clientKeyuserAuth
path Parameters
asset_type
required
string
Enum: "File" "Folder"

This is the type of asset you're trying to retrieve. Set to File if you're trying to get a file and Folder if you're trying to get a folder

asset_path
required
string

This either the asset's original path or a custom_path set when the asset was created.

Responses

A local demo server

http://localhost:5000/{asset_type}/{asset_path}
